## 1. 创建模型信息

### 2. 接口请求地址
`/source/model/create`

### 3. 请求方式
`POST`

### 4. 接口功能描述
创建新的大模型信息记录，保存模型的基本配置信息到数据库。

### 5. 请求头
| 参数名 | 参数值 | 是否必填 | 说明 |
|--------|--------|----------|------|
| Content-Type | application/json | 是 | 请求内容类型 |

### 6. 请求入参
| 参数名 | 参数类型 | 参数位置 | 是否必填 | 参数说明 | 示例值 |
|--------|----------|----------|----------|----------|--------|
| model | String | Body | 是 | 模型名称 | gpt-4 |
| baseUrl | String | Body | 是 | 模型 API 接口地址 | https://api.openai.com/v1 |
| apiKey | String | Body | 是 | API 密钥 | sk-xxxxxxxxx |
| manufacturerCode | String | Body | 是 | 模型厂商标识 | OPENAI |
| modelDescribe | String | Body | 否 | 模型描述信息 | GPT-4 大语言模型 |
| maxThreadSize | Long | Body | 否 | 最大并发数 | 10 |
| originName | String | Body | 否 | 环境参数标识 | production |
| maxCompletionTokens | Long | Body | 否 | 最大生成词元数量 | 2048 |
| stream | Boolean | Body | 否 | 是否流式响应，默认 false | false |
| config | Map<String, Object> | Body | 否 | 扩展配置项 | {"temperature": 0.7} |

### 7. 响应出参
| 字段名 | 字段类型 | 字段说明 | 示例值 |
|--------|----------|----------|--------|
| code | String | 状态码，0 表示成功 | 0 |
| message | String | 响应消息 | 操作成功 |
| requestId | String | 请求追踪 ID | req-123456 |
| data | Object | 响应数据 | - |
| ├─ id | Long | 模型主键 ID | 1 |
| ├─ model | String | 模型名称 | gpt-4 |
| ├─ baseUrl | String | 模型 API 接口地址 | https://api.openai.com/v1 |
| ├─ apiKey | String | API 密钥 | sk-xxxxxxxxx |
| ├─ manufacturerCode | String | 模型厂商标识 | OPENAI |
| ├─ modelDescribe | String | 模型描述信息 | GPT-4 大语言模型 |
| ├─ maxThreadSize | Long | 最大并发数 | 10 |
| ├─ originName | String | 环境参数标识 | production |
| ├─ maxCompletionTokens | Long | 最大生成词元数量 | 2048 |
| ├─ stream | Boolean | 是否流式响应 | false |
| ├─ config | Map<String, Object> | 扩展配置项 | {"temperature": 0.7} |
| ├─ version | Long | 版本号 | 1 |
| ├─ createTime | LocalDateTime | 创建时间 | 2026-05-01 12:00:00 |
| ├─ updateTime | LocalDateTime | 更新时间 | 2026-05-01 12:00:00 |

### 8. 成功响应 JSON 示例
```json
{
  "code": "0",
```

## 6. 大模型连通性测试

### 2. 接口请求地址
`/source/model/testConnectivity`

### 3. 请求方式
`POST`

### 4. 接口功能描述
测试大模型的连通性，验证模型配置是否正确可用。

### 5. 请求头
| 参数名 | 参数值 | 是否必填 | 说明 |
|--------|--------|----------|------|
| Content-Type | application/json | 是 | 请求内容类型 |

### 6. 请求入参
| 参数名 | 参数类型 | 参数位置 | 是否必填 | 参数说明 | 示例值 |
|--------|----------|----------|----------|----------|--------|
| model | String | Body | 是 | 模型名称 | gpt-4 |
| baseUrl | String | Body | 是 | 模型 API 接口地址 | https://api.openai.com/v1 |
| apiKey | String | Body | 是 | API 密钥 | sk-xxxxxxxxx |
| manufacturerCode | String | Body | 是 | 模型厂商标识 | OPENAI |

### 7. 响应出参
| 字段名 | 字段类型 | 字段说明 | 示例值 |
|--------|----------|----------|--------|
| code | String | 状态码，0 表示成功 | 0 |
| message | String | 响应消息 | 操作成功 |
| requestId | String | 请求追踪 ID | req-123456 |
| data | Object | 响应数据 | - |
| ├─ result | Boolean | 连通性测试结果，true 成功，false 失败 | true |
| ├─ respContent | String | 大模型返回的内容 | 连接成功 |
| ├─ elapsed | Long | 接口耗时（毫秒） | 500 |

### 8. 成功响应 JSON 示例
```json
{
  "code": "0",
  "message": "操作成功",
  "requestId": "req-123456",
  "data": {
    "result": true,
    "respContent": "连接成功",
    "elapsed": 500
  }
}
```

### 9. 失败响应 JSON 示例
```json
{
  "code": "500",
  "message": "不支持的模型厂商：UNSUPPORTED",
  "requestId": "req-123456",
  "data": null
}
```

### 10. 备注
- 支持的厂商枚举值：OPENAI、DEEPSEEK、QWEN、TELE、SPARK、GLM、KIMI
- manufacturerCode 需与枚举值一致（不区分大小写）
- 接口不依赖数据库，直接使用传入参数进行测试
