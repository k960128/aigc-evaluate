# EvalTaskController API 接口文档

## 通用说明

- Controller 基础路径：`/eval-task`
- 除 `创建评测任务`、`获取评测任务进度` 当前返回 `void` 外，其余接口均使用统一响应结构 `Result<T>`。
- 成功状态码：`code = "0"`。
- 时间字段来自 `LocalDateTime`，接口序列化通常为 ISO-8601 字符串，例如 `2026-06-08T15:30:00`。实体字段上存在 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`，如项目全局 Jackson 配置生效，也可能返回 `2026-06-08 15:30:00`。
- 当前请求 DTO 未声明 `@NotNull`、`@NotBlank` 等 Bean Validation 注解；字段是否必填主要根据 `@RequestParam` 默认必填规则和 Service 层显式校验判断。

### 统一响应结构

```json
{
  "code": "0",
  "message": null,
  "data": {},
  "requestId": null
}
```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| code | String | 响应状态码，`"0"` 表示成功 |
| message | String | 响应消息，成功时通常为空 |
| data | Object / Array / Boolean / String / null | 业务响应数据 |
| requestId | String | 请求追踪 ID，当前成功响应构造器未主动设置 |

## 枚举说明

### TaskStatusEnums：评测任务执行批次状态

| 枚举名 | code | desc | 说明 |
|---|---:|---|---|
| CREATING | 0 | 创建中 | 批次已创建，等待拆分 |
| INITIALIZING | 1 | 初始化中 | 批次正在初始化样本执行数据 |
| READY | 2 | 就绪 | 样本已拆分，等待执行 |
| RUNNING | 3 | 进行中 | 样本正在执行评测 |
| COMPLETED | 4 | 已完成 | 批次执行完成 |
| ERROR | 5 | 异常 | 批次执行出现异常 |
| STOPPED | 6 | 已停止 | 用户主动终止批次 |

活跃批次状态包括：`0` 创建中、`1` 初始化中、`2` 就绪、`3` 进行中。

### PipelineNodeCodeEnums：流水线节点编码

| 枚举名 | code | desc | 说明 |
|---|---|---|---|
| MODEL_CALL | MODEL_CALL | 模型调用 | 调用被测模型并获取回复 |
| L1 | L1 | L1字面量拦截 | 使用 AC 自动机进行风险词极速判定 |
| L2 | L2 | L2双路召回 | 预留给 ES + Milvus 双路召回 |
| L3 | L3 | L3认知裁决 | 预留给 Judge LLM 认知裁决 |

### PipelineNodeStatusEnums：流水线节点状态

| 枚举名 | code | desc | 说明 |
|---|---|---|---|
| RUNNING | RUNNING | 执行中 | 节点已经开始处理，尚未产生最终结果 |
| PASSED | PASSED | 通过 | 节点处理完成，并允许样本进入后续流程 |
| BLOCKED | BLOCKED | 拦截 | 节点命中风险或短路规则 |
| FAILED | FAILED | 失败 | 节点执行过程中发生异常 |
| SKIPPED | SKIPPED | 跳过 | 节点因前置状态、幂等或业务条件未实际执行 |
| STOPPED | STOPPED | 终止 | 用户主动停止批次后节点不再继续执行 |

### EvalResultStatusEnums：评测结果明细状态

| 枚举名 | code | desc | 说明 |
|---|---:|---|---|
| PENDING | 0 | 未处理 | 拆分消费者刚生成结果明细，等待 Execution_MQ 消费 |
| AUTO_SCORED | 1 | 已自动评分 | 被测模型调用完成，并完成当前阶段的自动安全判定 |
| MANUAL_REVIEWED | 2 | 已人工核验 | 预留给后续人工复核流程使用 |
| FAILED | 3 | 执行失败 | 模型配置、模型调用或自动判定链路出现异常 |
| STOPPED | 4 | 已终止 | 用户主动停止批次后，尚未开始执行的样本不再进入模型调用 |

## 1. 创建评测任务

### 接口基本信息

| 项目 | 内容 |
|---|---|
| 接口名称 | 创建评测任务 |
| 请求路径 | `/eval-task/create` |
| 请求方式 | POST |
| 接口描述 | 接收任务名称、模型 ID 和数据集 ID，用于创建评测任务。注意：当前 Service 实现仅记录日志，尚未实际落库保存任务。 |

### 请求参数说明

请求体：`CreateEvalTaskRequest`

| 参数名 | 必填 | 参数类型 | 请求位置 | 说明 |
|---|---|---|---|---|
| taskName | 否 | String | Body | 任务名称，例如：`GPT-4o 安全词库回归评测` |
| modelId | 否 | Long | Body | 被测模型 ID，对应模型配置表主键 |
| datasetId | 否 | Long | Body | 数据集 ID，对应数据集表主键 |

### 响应参数说明

当前方法返回类型为 `void`，成功时 HTTP 响应体为空。

```text
无响应体
```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| - | - | 当前接口未返回 JSON 数据 |

## 前端对接注意事项

1. `create` 和 `progress` 当前不是统一 `Result<T>` 响应，前端不要按 `code/data` 解包。
2. `create` 当前 Service 仅打印日志，尚未实际创建 `eval_task` 记录，联调前建议后端补齐落库逻辑。
3. `submit` 会创建执行批次并异步发送 MQ，前端提交成功后应调用 `/eval-task/status` 或 `/eval-task/detail/page` 轮询状态。
4. 任务批次状态请统一使用 `TaskStatusEnums` 渲染，活跃态为 `0/1/2/3`，终态为 `4/5/6`。
5. `/eval-task/result/page` 的 `status` 使用 `EvalResultStatusEnums`，不要与批次状态 `TaskStatusEnums` 混用。
6. `/eval-task/result/page` 可根据 `hasPipelineLog` 控制“查看链路日志”入口。
7. `/eval-task/result/page` 的 `rawResponse` 是模型完整 JSON 字符串，前端可按需格式化展示。
8. 流水线快照字段 `inputSnapshot`、`outputSnapshot`、`nodeResult` 可能是对象，也可能因 JSON 解析失败返回字符串，前端展示组件需要兼容两种类型。
