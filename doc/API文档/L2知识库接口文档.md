# L2KnowledgeBaseController API 接口文档

## 通用说明

- Controller 基础路径：`/risk/l2/kb`
- 所有接口均使用统一响应结构 `Result<T>`。
- 成功状态码：`code = "0"`。
- 时间字段来自 `LocalDateTime`，实体字段上存在 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`，通常返回 `2026-06-13 15:30:00` 形式；如全局 Jackson 配置覆盖，也可能返回 ISO-8601 字符串。
- 当前 Controller 未使用 `@Validated`，请求 DTO 字段也未声明 `@NotNull`、`@NotBlank` 等 Bean Validation 注解；Body 内字段按源码校验结果标注为非必填。`@PathVariable` 和 `@RequestParam` 参数按 Spring 默认规则标注为必填。
- 分页接口当 `current` 或 `size` 为空、或小于等于 0 时，后端分别使用默认值 `current = 1`、`size = 10`。
- 创建、更新、启禁用和删除接口当前返回 `Boolean`，表示 MyBatis-Plus 写操作是否成功，不返回变更后的实体详情。

### 统一响应结构

```json
{
  "code": "0",
  "message": null,
  "data": {},
  "requestId": null
}
```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| code | String | 响应状态码，`"0"` 表示成功 |
| message | String | 响应消息，成功时通常为空 |
| data | Object / Array / Boolean / null | 业务响应数据 |
| requestId | String | 请求追踪 ID，当前成功响应构造器未主动设置 |

### 枚举和约定

| 字段 | 可选值 | 说明 |
|---|---|---|
| severityLevel / riskLevel | `1`、`2`、`3`、`4` | 风险等级：1-低，2-中，3-高，4-致命 |
| status | `0`、`1` | 业务状态：0-禁用，1-启用 |
| featureType | `KEYWORD`、`PAYLOAD`、`PROMPT_PATTERN`、`RESPONSE_PATTERN`、`JAILBREAK`、`SIMILAR_CASE`、`SAFE_EXCEPTION` | 攻击特征类型 |
| polarity | `UNSAFE`、`SAFE_EXCEPTION` | 特征极性：风险特征或安全例外 |
| syncStatus | `0`、`1`、`2`、`3` | 综合同步状态：0-待同步，1-已同步，2-同步失败，3-已删除待同步 |
| esSyncStatus / milvusSyncStatus / pgSyncStatus | `0`、`1`、`2` | 单索引同步状态：0-待同步，1-已同步，2-同步失败 |
| esStatus / milvusStatus / pgStatus | `0`、`1`、`2` | 同步事件处理状态：0-待处理，1-成功，2-失败 |
| aggregateType | `ATTACK_FEATURE`、`DETAIL_RULE` | 同步事件聚合类型 |
| operationType | `CREATE`、`UPDATE`、`DELETE`、`REINDEX` | 同步事件操作类型 |

### 分页响应结构

分页接口返回 MyBatis-Plus `Page<T>`。

```json

  "records": [],

  "size": 10,

  "pages": 0

```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| records | Array | 当前页记录列表 |
| total | Long | 总记录数 |
| size | Long | 每页数量 |
| current | Long | 当前页码 |
| pages | Long | 总页数 |

## 1. 分页查询 L2 风险小类判定规则

### 接口基本信息

| 项目 | 内容 |
|---|---|
| 接口名称 | 分页查询 L2 风险小类判定规则 |
| 请求路径 | `/risk/l2/kb/rules/page` |
| 请求方式 | POST |
| 接口描述 | 按风险小类、严重等级、业务状态和规则关键词分页查询 `risk_detail_rule`。结果按 `riskDetailsId` 升序、`updateTime` 降序返回。 |

| 参数名 | 必填 | 参数类型 | 请求位置 | 说明 |

| keyword | 否 | String | Body | 判定规则关键词，模糊查询 `judgeRule` |

## 前端对接注意事项

1. 规则接口和特征接口的创建、更新、删除均返回 `Boolean`，前端如需展示最新详情，需要在操作成功后再调用详情或分页接口刷新数据。
2. 创建规则时后端固定设置 `version = 1`、`status = 1`；创建特征时后端固定设置 `version = 1`、`status = 1`，并将各同步状态置为待同步。
3. 更新特征和启禁用特征会将 `syncStatus`、`esSyncStatus`、`milvusSyncStatus`、`pgSyncStatus` 重置为 `0`，前端可据此提示后续索引同步状态。
4. `/features/page` 当前支持按 `esSyncStatus`、`pgSyncStatus` 筛选，但未提供 `milvusSyncStatus` 入参筛选。
5. `/sync-events/page` 当前支持按 `esStatus`、`pgStatus` 筛选，但未提供 `milvusStatus` 入参筛选。
6. 当前请求 DTO 未做 Bean Validation，参数缺失可能不会在 Controller 层拦截，但仍可能触发数据库非空、唯一索引或类型约束错误。
7. `unsafeExamples`、`safeExamples`、`tags`、`payload` 在 Java 实体中均表现为 `String`，业务内容建议传 JSON 字符串。
