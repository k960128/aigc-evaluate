# DataSetController API 接口文档

## 通用说明

- Controller 基础路径：`/data-set`
- 所有接口均使用统一响应结构 `Result<T>`。
- 成功状态码：`code = "0"`。
- 时间字段来自 `LocalDateTime`，接口序列化通常为 ISO-8601 字符串，例如 `2026-06-08T15:30:00`。实体字段上存在 `@JsonFormat(pattern = "yyyy-MM-dd HH:mm:ss")`，如项目全局 Jackson 配置生效，也可能返回 `2026-06-08 15:30:00`。
- 当前请求 DTO 未声明 `@NotNull`、`@NotBlank` 等 Bean Validation 注解；Body 内字段按源码校验结果标注为非必填，`@RequestParam` 参数按 Spring 默认规则标注为必填。
- `datasetType` 当前源码中仅定义为 `Integer` 类型，未发现对应数据集类型枚举；文档仅按“评测/业务类型编码”说明，不臆造枚举值。

### 变更说明

- `dataset_sample.input_text` 已变更为 `question`，数据集样本接口响应字段同步由 `inputText` 调整为 `question`。
- `dataset_sample.dataset_id` 为 `bigint`，接口响应中的 `datasetId` 类型为 `Long`。
- 新表结构不再包含 `answerText`、`scoreRule`、`field`，数据集样本接口不再返回这些字段。

### 统一响应结构

```json
{
  "code": "0",
  "message": null,
  "data": {},
  "requestId": null
}
```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| code | String | 响应状态码，`"0"` 表示成功 |
| message | String | 响应消息，成功时通常为空 |
| data | Object / Array / null | 业务响应数据 |
| requestId | String | 请求追踪 ID，当前成功响应构造器未主动设置 |

## 1. 创建数据集

### 接口基本信息

| 项目 | 内容 |
|---|---|
| 接口名称 | 创建数据集 |
| 请求路径 | `/data-set/create` |
| 请求方式 | POST |
| 接口描述 | 创建一条数据集基础信息记录，包含数据集名称、类型、样本数量和描述信息。 |

### 请求参数说明

请求体：`CreateDataSetRequest`

| 参数名 | 必填 | 参数类型 | 请求位置 | 说明 |
|---|---|---|---|---|
| datasetName | 否 | String | Body | 数据集名称，例如：`中文安全指令数据集` |
| datasetType | 否 | Integer | Body | 数据集类型，用于标识评测数据集所属的业务类型；当前源码未定义枚举 |
| sampleCount | 否 | Integer | Body | 样本数量，表示当前数据集中包含的样本总数 |
| description | 否 | String | Body | 数据集描述，用于补充说明数据集用途、来源或适用场景 |

### 响应参数说明

```json
{
  "code": "0",
  "message": null,
  "data": {
    "id": 3001,
    "datasetName": "中文安全指令数据集",
    "datasetType": 1,
    "sampleCount": 120,
    "description": "用于内容安全、拒答能力和高危指令识别的中文评测样本集",
    "createTime": "2026-06-08T15:10:00",
    "updateTime": "2026-06-08T15:10:00"
  },
  "requestId": null
}
```

| 字段名 | 数据类型 | 说明 |
|---|---|---|
| data.id | Long | 数据集 ID，对应数据集信息表主键 |
| data.datasetName | String | 数据集名称 |
| data.datasetType | Integer | 数据集类型编码，当前源码未定义枚举 |
| data.sampleCount | Integer | 样本数量 |
| data.description | String | 数据集描述 |
| data.createTime | String | 创建时间 |
| data.updateTime | String | 更新时间 |

## 2. 更新数据集

| 项目 | 内容 |
|---|---|
| 接口名称 | 更新数据集 |
| 请求路径 | `/data-set/update` |
| 请求方式 | PUT |
| 接口描述 | 根据数据集 ID 更新数据集基础信息；数据集不存在时返回成功响应但 `data = null`。 |

### 请求参数说明

请求体：`UpdateDataSetRequest`

| 参数名 | 必填 | 参数类型 | 请求位置 | 说明 |
|---|---|---|---|---|
| id | 否 | Long | Body | 数据集 ID。业务上需要传入，但源码未加 `@NotNull` 校验 |
| datasetName | 否 | String | Body | 数据集名称 |
| datasetType | 否 | Integer | Body | 数据集类型，用于标识评测数据集所属的业务类型；当前源码未定义枚举 |
| sampleCount | 否 | Integer | Body | 样本数量，表示当前数据集中包含的样本总数 |
| description | 否 | String | Body | 数据集描述，用于补充说明数据集用途、来源或适用场景 |

### 响应参数说明

```json
{
  "code": "0",
  "message": null,
  "data": {
    "id": 3001,
    "datasetName": "中文安全指令数据集 v2",
    "datasetType": 1,
    "sampleCount": 150,
    "description": "补充越狱提示、内容安全和高危操作拒答样本后的回归评测数据集",
    "createTime": "2026-06-08T15:10:00",
    "updateTime": "2026-06-08T16:05:20"
  },
  "requestId": null
}
```

响应字段同“创建数据集”。数据集不存在时：

```json
{
  "code": "0",
  "message": null,
  "data": null,
  "requestId": null
}
```

| 项目 | 内容 |
|---|---|
| 接口名称 | 分页查询数据集样本列表 |

| 请求方式 | POST |

| 字段名 | 数据类型 | 说明 |
|---|---|---|

## 前端对接注意事项

1. `datasetType` 当前没有后端枚举定义，前端如需展示文案，应以产品或配置侧约定为准，不要从本接口文档推导固定枚举。
2. `/data-set/sample/page` 的 `datasetId` 非必填；不传时会分页返回全部样本。
3. `/data-set/sample/list` 的 `datasetId` 是 Query 必填参数，适合在已选定某个数据集后加载全部样本。
4. 更新数据集时，如果 `id` 对应数据不存在，接口返回成功响应且 `data = null`，前端需要兼容空数据。
