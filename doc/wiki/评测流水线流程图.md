# 评测流水线流程图

本文档描述当前后端评测流水线的完整执行路径，覆盖任务提交、批次拆分、单样本执行、L1/L2 安全判定、批次进度推进、用户停止和流水线节点日志。

## 总览流程

```mermaid
flowchart TD
    A["用户提交评测任务<br/>EvalTaskController.submitEvalTask"] --> B["EvalTaskServiceImpl.submitEvalTask"]
    B --> C{"是否存在活跃批次<br/>CREATING / INITIALIZING / READY / RUNNING"}
    C -- "是" --> C1["拒绝重复提交<br/>抛出业务异常"]
    C -- "否" --> D["校验任务、模型、数据集样本数量"]
    D --> E["创建 eval_task_detail<br/>status = CREATING<br/>finished_count = 0<br/>failed_count = 0"]
    E --> F["发送 EvalTaskSplit_MQ<br/>EvalTaskMqProducer.sendTaskSplitMessage"]

    F --> G["EvalTaskSplitConsumer 消费拆分消息"]
    G --> H{"批次状态"}
    H -- "READY / RUNNING" --> H1["补发仍为 PENDING 的样本执行消息<br/>防止拆分消息重试或投递遗漏"]
    H -- "COMPLETED / ERROR / STOPPED" --> H2["终态批次<br/>跳过重复拆分"]
    H -- "CREATING" --> I["批次置为 INITIALIZING<br/>记录 start_time"]

    I --> J["加载数据集样本 data_set_sample"]
    J --> K{"样本是否为空"}
    K -- "是" --> K1["批次置为 ERROR<br/>结束拆分"]
    K -- "否" --> L["为每个样本创建 eval_result_detail<br/>status = PENDING"]
    L --> M{"拆分期间是否被用户停止"}
    M -- "是" --> M1["停止投递 Execution_MQ"]
    M -- "否" --> N["批次置为 READY<br/>更新 total_count"]
    N --> O["逐条发送 Execution_MQ<br/>EvalSampleExecutionMessage"]

    O --> P["EvalSampleExecutionConsumer 消费单样本"]
    P --> Q["执行 MODEL_CALL / L1 / L2<br/>详见单样本流程图"]
    Q --> R["写入 eval_result_detail 终态<br/>推进 finished_count / failed_count"]
    R --> S{"finished_count 是否达到 total_count"}
    S -- "否" --> S1["批次保持 RUNNING"]
    S -- "是且 failed_count = 0" --> S2["批次置为 COMPLETED"]
    S -- "是且 failed_count > 0" --> S3["批次置为 ERROR"]
```

## 单样本执行流程

```mermaid
flowchart TD
    A["Execution_MQ 消息<br/>resultDetailId / taskDetailId / modelId"] --> B["按 resultDetailId 获取 Redisson 锁<br/>防止同一样本并发执行"]
    B --> C{"是否拿到锁"}
    C -- "否" --> C1["跳过重复消息"]
    C -- "是" --> D["查询 eval_result_detail"]
    D --> E{"结果是否已是终态<br/>AUTO_SCORED / MANUAL_REVIEWED / FAILED / STOPPED"}
    E -- "是" --> E1["跳过重复执行"]
    E -- "否" --> F["查询 eval_task_detail"]
    F --> G{"批次是否 STOPPED"}
    G -- "是" --> G1["记录 MODEL_CALL 节点 STOPPED<br/>不调用模型"]
    G -- "否" --> H["批次 READY -> RUNNING<br/>仅 READY 可推进"]

    H --> I["开始 MODEL_CALL 节点<br/>写 eval_pipeline_node_detail RUNNING"]
    I --> J["调用被测模型<br/>ModelClientStrategy.call"]
    J --> K{"模型调用是否成功"}
    K -- "否" --> K1["MODEL_CALL 节点 FAILED<br/>样本 FAILED<br/>failed_count + 1"]
    K -- "是" --> L["MODEL_CALL 节点 PASSED<br/>保存模型输出和耗时"]

    L --> M{"模型调用后批次是否 STOPPED"}
    M -- "是" --> M1["记录 L1 节点 STOPPED<br/>丢弃模型返回结果"]
    M -- "否" --> N["开始 L1 节点<br/>AC 自动机字面量风险判定"]

    N --> O{"L1 是否命中拦截"}
    O -- "是" --> O1["L1 节点 BLOCKED<br/>is_safe = false<br/>status = AUTO_SCORED"]
    O -- "否" --> P["L1 节点 PASSED<br/>携带 warningTags 进入 L2"]

    P --> Q{"L1 后批次是否 STOPPED"}
    Q -- "是" --> Q1["丢弃本次结果<br/>不推进最终态"]
    Q -- "否" --> R["开始 L2 节点<br/>双路召回 + RRF + Reranker + 路由"]

    R --> S{"L2 决策"}
    S -- "UNSAFE<br/>高置信违规" --> S1["L2 节点 BLOCKED<br/>is_safe = false<br/>status = AUTO_SCORED"]
    S -- "SAFE / DEGRADED_SAFE<br/>安全或降级安全" --> S2["L2 节点 PASSED<br/>is_safe = true<br/>status = AUTO_SCORED"]
    S -- "PASS_TO_L3<br/>模糊区" --> S3["L2 节点 PASSED<br/>is_safe = null<br/>status = MANUAL_REVIEWED"]
    R --> T{"L2 是否异常"}
    T -- "是" --> T1["L2 节点 FAILED<br/>样本 FAILED<br/>failed_count + 1"]

    O1 --> U["成功结果事务落库<br/>finished_count + 1"]
    S1 --> U
    S2 --> U
    S3 --> U
    K1 --> V["失败结果事务落库<br/>finished_count + 1<br/>failed_count + 1"]
    T1 --> V
```

## L2 内部判定流程

```mermaid
flowchart TD
    A["L2EvaluationService.evaluate"] --> B["构造查询文本<br/>用户输入 + 模型输出 + L1 warningTags"]
    B --> C["生成 queryTextDigest<br/>流水线日志只保存摘要"]
    C --> D["L2RecallClient.recall"]

    D --> E{"当前召回实现"}
    E -- "MySQL Mock" --> E1["读取 risk_attack_feature<br/>轻量文本匹配<br/>生成模拟 ES / Milvus 命中"]
    E -- "空降级" --> E2["返回空命中<br/>degraded = true"]
    E1 --> F["ES hits + Milvus hits"]
    E2 --> F

    F --> G["RRF 融合<br/>按 featureId 合并两路来源"]
    G --> H["Reranker 精排<br/>当前默认优先沿用 Mock rerankScore<br/>否则使用 RRF 分"]
    H --> I["按 riskDetailsId 聚合"]
    I --> J["计算 detailScore<br/>最高 rerank 分<br/>+ UNSAFE 数量权重<br/>+ 强 ES 命中<br/>+ 高 Milvus 相似<br/>+ 风险等级<br/>- SAFE_EXCEPTION 扣分"]

    J --> K{"是否有风险小类命中"}
    K -- "否，且 degraded = true" --> K1["DEGRADED_SAFE<br/>降级安全"]
    K -- "否，且 degraded = false" --> K2["SAFE<br/>低风险安全"]
    K -- "有命中" --> L{"是否满足高置信违规"}
    L -- "是" --> L1["UNSAFE<br/>detailScore 超阈值<br/>或单条强证据命中"]
    L -- "否" --> M{"是否满足低风险安全"}
    M -- "是" --> M1["SAFE<br/>低分或无 UNSAFE 命中"]
    M -- "否" --> M2["PASS_TO_L3<br/>L3 暂未接入<br/>进入人工核验"]

    K1 --> N["构造 output_snapshot / node_result"]
    K2 --> N
    L1 --> N
    M1 --> N
    M2 --> N
```

## 状态流转

### 批次状态 eval_task_detail.status

```mermaid
stateDiagram-v2
    [*] --> CREATING: submitEvalTask 创建批次
    CREATING --> INITIALIZING: TaskSplitConsumer 接手拆分
    INITIALIZING --> READY: 样本结果明细创建完成
    READY --> RUNNING: 第一个样本开始执行
    RUNNING --> COMPLETED: 全部样本完成且 failed_count = 0
    RUNNING --> ERROR: 全部样本完成且 failed_count > 0
    CREATING --> ERROR: 拆分消息发送失败
    INITIALIZING --> ERROR: 数据集为空或拆分异常
    CREATING --> STOPPED: 用户停止
    INITIALIZING --> STOPPED: 用户停止
    READY --> STOPPED: 用户停止
    RUNNING --> STOPPED: 用户停止
```

### 样本状态 eval_result_detail.status

```mermaid
stateDiagram-v2
    [*] --> PENDING: TaskSplitConsumer 创建结果明细
    PENDING --> AUTO_SCORED: L1 拦截 / L2 UNSAFE / L2 SAFE
    PENDING --> MANUAL_REVIEWED: L2 PASS_TO_L3 模糊区
    PENDING --> FAILED: 模型调用、L1、L2 执行异常
    PENDING --> STOPPED: 用户停止且样本尚未执行
```

## 停止与幂等控制

```mermaid
flowchart TD
    A["用户停止任务<br/>EvalTaskController.stopEvalTask"] --> B["EvalTaskServiceImpl.stopEvalTask"]
    B --> C["获取任务操作锁<br/>按 taskId"]
    C --> D{"是否存在活跃批次"}
    D -- "否" --> D1["直接返回成功"]
    D -- "是" --> E["批次置为 STOPPED<br/>写 end_time"]
    E --> F["仍为 PENDING 的样本置为 STOPPED"]

    G["Execution_MQ 重复投递"] --> H["按 resultDetailId 获取执行锁"]
    H --> I{"结果是否终态"}
    I -- "是" --> I1["跳过重复执行"]
    I -- "否" --> J["继续执行"]

    K["EvalTaskSplit_MQ 重复投递"] --> L{"批次状态"}
    L -- "READY / RUNNING" --> L1["只补发 PENDING 样本"]
    L -- "终态" --> L2["跳过"]
    L -- "CREATING" --> L3["继续拆分"]
```

## 当前阶段边界

- L1 已接入 AC 自动机字面量拦截。
- L2 已接入召回、RRF、Reranker 降级和阈值路由。
- 当前 MySQL Mock 召回用于无 ES/Milvus 数据时验证链路。
- L3 Judge LLM 暂未实现，L2 `PASS_TO_L3` 结果会进入人工核验。
- `eval_pipeline_node_detail` 是排查单样本执行过程的主要日志表。
